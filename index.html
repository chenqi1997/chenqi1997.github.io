90. Subsets II/子集 II

题目：

给定一个可能包含重复元素的整数数组 nums，返回该数组所有可能的子集（幂集）。

说明：解集不能包含重复的子集。

示例:

解法一：DFS+剪枝回溯

这个题跟前面的 Leetcode 78 子集 总体思路是一样的，只不过加上了可能重复的元素，面对这样的条件也是很套路化了，先 sort 一遍，然后按照正常的回溯法来依次加到 tmp 里，再把 tmp 依次加到 res 里。

⭐⭐⭐整个代码最关键的一行在于 if(i > num && nums[i] == nums[i - 1]) continue;

这一行里面重点在于 i > num 这一个剪枝判断方法，这一个的目的是可以用 num 作为区分重复元素应不应该放进 tmp 的标准。i > num 就 continue 可以保证第一个 i = num 的时候可以取到重复元素的第一个元素并正常push 作为头节点。也就是说剪枝是从每一个递归层的第二层开始的，所以加上 i > num 。

  2. 给定矩阵中至少有一个元素是 0。
  3. 矩阵中的元素只在四个方向上相邻: 上、下、左、右。

解法一：动态规划

最开始想到的方法是动态规划，不过这里面题目中每一个点的值其实是取值与上下左右四个点的，而一般的动态规划只能够从左上到右下进行一次遍历，这样得到的结果是缺少了右侧和下方的数值的，因此可以想到用两次动态规划遍历同一个 dp 数组，第二次从右下向左上遍历。

解法二：BFS

相当于把所有的 0 看成是一个整体，存在一个 超级零 与所有的0相连，相当于最短路算法中的 超级远点 ，连向所有的源点，用 超级源点 到终点的最短路等价多个源点到终点的终点。在起始的时候把所有的位置为 0 的点都放入队列中，然后不断广播更新 result 二维矩阵中的值：如果当前广播到的 result[x][y] 的值大于当前队首坐标的值 + 1 ，那么就可以用这个队首坐标的值 + 1来代替当前的 result。同时将这个目前广播到的(x, y) 入队，进行更深层次的广播。

解法：找规律

解法一：暴力递归法

这道题看了一下 dp 解法。好像并没有比递归简单很多，很多人给的 dp 法空间复杂度 O(N^3)，时间复杂度O(N^4)。琢磨了一下主要是因为递归可以对很多不符合的情况直接 return ，而动态规划需要依赖之前的状态一直计算下去。

你应当保留两个分区中每个节点的初始相对位置。

示例:

解法：⭐⭐⭐简单链表+双指针

本来是10分钟能做完的，可惜写错了一点拖到了20分钟。这里面交换的思路很简单，就是使用两个指针 pre 和 cur ，分别保存从前到后第一个出现的大于等于 x 的节点的上一个节点，和当前遍历到的节点。然后如果当前遍历到的节点 cur 的下一个节点值小于 x ，则需要进行调换操作。注意，如果调换了的情况下 cur 是不需要右移的，如果没调换的话 cur 才需要右移。第一次解法弄错了这里。

这里面还有一些需要注意的corner problem是比如头节点就比 x 大，因此我们需要使用一个哨兵节点 dummy 指向头节点，同时还有最后一个节点的情况也需要注意。

题目：

给定一个仅包含 0 和 1 的二维二进制矩阵，找出只包含 1 的最大矩形，并返回其面积。

示例:

解法：⭐⭐⭐⭐⭐单调栈+dp

这道题用到了跟上一题 Leetcode 84 相同的单调栈思想，如果用单纯的暴力解法这道题需要的时间是 O(N^3)，可以想到利用上一题的函数实现，在对每一行的遍历的过程中维护一个一维 dp 数组，记录的是当前行当前列元素上面柱形的高度是多少，状态转移方程为： dp[j] = matrix[i][j] == '1' ? dp[j] + 1 : 0

然后再在每一层的递归当中不断调用 largestRectangleArea() 函数来更新 maxarea 的值。最终的时间复杂度为 O(MN) ，空间复杂度为 O(M) 。

84*. Largest Rectangle in Histogram/柱状图中最大的矩形

题目:

给定 n 个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为 1 。

求在该柱状图中，能够勾勒出来的矩形的最大面积。

示例:

解法一：两次遍历笨法（超时）

不过 O(N^2) 时间复杂度还是通过了 94/96 个样例hhh

解法二：⭐⭐⭐⭐⭐单调栈

很巧妙，类似的单调栈问题还有 Leetcode 42/496/503/739/901 。

我们发现了，只要是遇到了当前柱形的高度比它上一个柱形的高度严格小的时候，一定可以确定它之前的某些柱形的最大宽度，并且确定的柱形宽度的顺序是从右边向左边。
这个现象告诉我们，在遍历的时候需要记录的信息就是遍历到的柱形的下标，它一左一右的两个柱形的下标的差就是这个面积最大的矩形对应的最大宽度。

单调栈特别适合解决那些，两头大小决定中间值的大小的题。

因为单调栈中，以单调增栈为例。
设栈顶元素为 b， 栈顶第二个元素为a，自然有 a < b （因为堆栈越往上越大）
这时候， 若c出现，且c小于b， 那么b的左右第一个比b小的两个元素就找到了，分别是a和c，b在中间最大。
这时候你可以处理b，并重新整理堆栈，使其保持递增;若c大于b，那c入栈，继续循环就行了。
最后清理堆栈

思路：

  1. 对于一个下标对应的高度，如果能得到向左和向右的边界
  2. 那么就能对每个高度求一次面积
  3. 遍历所有高度，即可得出最大面积
