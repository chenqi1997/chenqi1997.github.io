示例 2:

解法：⭐⭐⭐⭐⭐摩尔投票法

这道题跟 Leetcode 169 多数元素 差不多，只不过是169题里面只需要，找一个众数，那就只要一个候选，并且是保证一定有一个是众数的，直接投票就好；但是这个题没有保证有一个元素一定出现 n/3 以上，因此我们需要两次遍历。

摩尔投票法

摩尔投票法分为两个阶段：抵消阶段和计数阶段。

抵消阶段：两个不同投票进行对坑，并且同时抵消掉各一张票，如果两个投票相同，则累加可抵消的次数；

计数阶段：在抵消阶段最后得到的抵消计数只要不为 0 ，那这个候选人是有可能超过一半的票数的，为了验证，则需要遍历一次，统计票数，才可确定。

首先对于这样的问题，数学角度来看，个数超过n/k 的众数最多只有 k - 1 个。

因此我们可以在这道题里面先指定两个候选人 cand1 ，cand2 。然后不断遍历整个数组：

  1. 如果投 A （当前元素等于 A ），则 A 的票数++;
  2. 如果投 B （当前元素等于 B ）， 则 B 的票数++；
  3. 如果 A , B 都不投（即当前与 A ， B 都不相等）,那么检查此时 A 或 B 的票数是否减为 0 ，如果为 0 ，则当前元素成为新的候选人；如果 A , B 两个人的票数都不为 0 ，那么 A , B 两个候选人的票数均减一。此时说明是另外一个候选人，需要分别抵消掉 A 和 B 的一张选票。

这样抵消遍历投票一个循环下来，最后得到的两个候选人就是有可能投票超过 n / 3 的候选人，但是还需要一次计数遍历来判断是否投票真的超过了 n / 3 。

解法：⭐⭐⭐⭐⭐利用平衡二叉搜索树BST+Set实现

set 是 STL 中一种标准关联容器。它底层使用平衡的搜索树——红黑树实现，插入删除操作时仅仅需要指针操作节点即可完成，不涉及到内存移动和拷贝，所以效率比较高。set ，顾名思义是“集合”的意思，在 set 中元素都是唯一的，而且默认情况下会对元素自动进行升序排列，支持集合的交 (set_intersection) , 差 (set_difference) 、 并 (set_union) ，对称差 (set_symmetric_difference) 等一些集合上的操作，如果需要集合中的元素允许重复那么可以使用 multiset 。

总体思想：

  1. 初始化一颗空的二叉搜索树 set。
  2. 对于每个元素 x ，遍历整个数组。
  3. 在 set 上查找大于等于 x 的最小的数，如果s − x ≤ t 则返回 true 。
  4. 在 set 上查找小于等于 x 的最大的数，如果 x − g ≤ t 则返回 true 。
  5. 在 set 中插入x ，如果树的大小超过了 k , 则移除最早加入树的那个数。
  6. 返回 false

这里面 C++ 的 set::upper_bound和 Java 中的TreeSet::floor有点微妙的不同。为了遵循区间的左闭右开原则，set::upper_bound返回的是小于等于目标元素的最大元素的后一个元素（即大于目标元素的最小元素），所以要对返回的迭代器进行自减操作。

简单来说就是：

std::set::upper_bound 是 >
std::set::lower_bound 是 >=

解法二：⭐⭐⭐⭐⭐桶排序

我们换种思路。 由于本题对索引有要求，因此直接排序破坏了原来的索引的做法是不行的。 我们考虑使用桶排序。我们把窗口内的数字存在不同编号的桶中。每个桶内存的数字范围是 t + 1 个数，这样做的好处是，桶内任意两个数之间的差一定是小于等于 t 的。

  1. 我们将数据分到 M 个桶 中。
  2. 每个数字 nums[i] 都被我们分配到一个桶中。
  3. 分配的依据就是 nums[i] / (t + 1)。
  4. 这样相邻桶内的数字最多相差 2 * t + 1。
  5. 不相邻的桶一定不满足相差小于等于 t 。
  6. 同一个桶内共有 t + 1 个下标连续的数字，数字最多相差 t。
  7. 因此如果命中同一个桶内，那么直接返回 True 。
  8. 如果命中相邻桶，我们再判断一下是否满足 相差 <= t。
  9. 否则返回 False 。

需要注意的是，由于题目有索引相差 k 的要求，因此要维护一个大小为 k 的窗口，定期清除桶中过期的数字。在具体实现中使用哈希表来模拟桶， key 就是桶号，value 就是数字本身。时间复杂度为 O(N) 。

560. Subarray Sum Equals K/和为K的子数组

题目：

给定一个整数数组和一个整数 k，你需要找到该数组中和为 k 的连续的子数组的个数。

示例 1 :

说明 :

  1. 数组的长度为 [1, 20,000]。
  2. 数组中元素的范围是 [-1000, 1000] ，且整数 k 的范围是 [-1e7, 1e7]。

解法一：两次遍历 O(N^2)时间复杂度

解法二：⭐⭐⭐⭐空间优化时间 哈希表

我们定义pre[i] 为 [0..i] 里所有数的和，则 pre[i] 可以由 pre[i−1] 递推而来，即：

$$
pre[i]=pre[i−1]+nums[i]
$$
那么[j..i] 这个子数组和为 k ，这个条件我们可以转化为

$$
pre[i]−pre[j−1]==k
$$

简单移项可得符合条件的下标 j 需要满足

$$
pre[j−1]==pre[i]−k
$$
所以我们考虑以 i 结尾的和为 k 的连续子数组个数时只要统计有多少个前缀和为 pre[i] − k的 pre[j] 即可。我们建立哈希表 map ，以当前各个位置的数字和为键，出现这个数字和的次数为对应的值，记录 pre[i] 出现的次数，从左往右边更新 map 边计算答案，那么以 i 结尾的答案 map[pre[i]−k] 即可在 O(1) 时间内得到。最后的答案即为所有下标结尾的和为 k 的子数组个数之和。

这里面边缘条件是要加入一个 map[0] = 1 的映射。
