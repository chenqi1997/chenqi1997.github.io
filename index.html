解法一：循环+回溯（部分样例超时）

解法二：⭐⭐⭐记忆化回溯

拿 [aaaab] 来举例，它可分为 ["a","a","aab"] , ["aa","aab"] , 两个都包含了 "aab" ,所以就产生了重复的
递归。这时，就可以用一个哈希表记录 "aab" 这种情况。当再次递归时，先判断一下，如果是 "aab" ,就直接返回。

解法三：⭐⭐⭐⭐动态规划+贪心思想

  1. 定义 vector<bool> dp(s.size() + 1, false)
  2. dp[i] 表示第 i 个字符前，所有的字符是否可以按字典拆分成单词；其中第 i - 1 个字符是上一个词的结尾字母，第 i 个字符是一个新词的开头字母。
  3. 初始值 dp[0] = true ，表示第 0 个字符是一个新词的开头字母；所求的答案为 dp[s.size()] ，表示这个字符串是否可以拆分成字典中的单词。
  4. 从前往后遍历字符串，我们只对新词的开头字母进行字典匹配，将每一个匹配成功的词后面的字母更新为新词开头字母。
  5. 设置一个变量记录所有最大的新词开头字母，如果当前字符已经超过了所有新词开头字母，可以剪枝返回 false 了。

这个解法里面用到了 C++ 的 memcmp 库函数：

库函数 int memcmp(const void *str1, const void *str2, size_t n)) 把存储区 str1 和存储区 str2 的前 n 个字节进行比较。

  • str1 – 指向内存块的指针。
  • str2 – 指向内存块的指针。
  • n – 要被比较的字节数。
  • 如果返回值 < 0，则表示 str1 小于 str2。
  • 如果返回值 > 0，则表示 str2 小于 str1。
  • 如果返回值 = 0，则表示 str1 等于 str2。

解法一：最笨的方法

本来以为这个肯定超时或者overflow，没想到居然过了❓

解法二：官方贪心算法

官方的答案不说人话，简单翻译一下就是，这道题的贪心算法在于：

如果A站不能到B站，那么A，B之间到任何一个站都不能到B站，（B站是A站第一个不能到的站）

因为在从 A 站遍历到 B 站的过程中，反证假如 A , B 中间某个 C 站可以到达 B 站，那么 A 能走到 C 说明 A - C 的 curr_tank > 0 ， C 能走到 B 说明 C - B 的 curr_tank > 0 ，与 A - B 的 curr_tank < 0 矛盾。

因此基于这个假设，我们可以直接在 curr_tank < 0 的时候果断把开始的加油站换为 i + 1。如果最终遍历下来 total_tank 是大于 0 的，并且在 i 到结尾这段路中间全都有 curr_tank >= 0，可以证明总油量大于总路程的情况下，必有解。

1095. Find in Mountain Array/山脉数组中查找目标值

题目：

（这是一个 交互式问题 ）

给你一个 山脉数组 mountainArr，请你返回能够使得 mountainArr.get(index) 等于 target 最小 的下标 index 值。

如果不存在这样的下标 index，就请返回 -1。

何为山脉数组？如果数组 A 是一个山脉数组的话，那它满足如下条件：

首先，A.length >= 3

其次，在 0 < i < A.length - 1 条件下，存在 i 使得：

  • A[0] < A[1] < ... A[i-1] < A[i]
  • A[i] > A[i+1] > ... > A[A.length - 1]

你将 不能直接访问该山脉数组，必须通过 MountainArray 接口来获取数据：

  • MountainArray.get(k) - 会返回数组中索引为k 的元素（下标从 0 开始）
  • MountainArray.length() - 会返回该数组的长度

注意：

对 MountainArray.get 发起超过 100 次调用的提交将被视为错误答案。此外，任何试图规避判题系统的解决方案都将会导致比赛资格被取消。
