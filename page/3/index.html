题目：

示例 2:

解法：⭐⭐⭐⭐⭐两次循环+哈希表

这道题其实思想并不难，难的是如何控制时间并且考虑好边缘情况。可能出现的边缘情况如下。

  1. 点重合
  2. 斜率为 0 或 $\infty $ 。
  3. 后来遍历的点在之前的直线上已经遍历过。
  4. 存在哈希表中的斜率精度难以满足。

其中尤其以第 4 个问题难以解决。测试样例中有 [[0,0],[94911150,94911151],[94911151,94911152]] 这个样例会被判断为在一个直线上，使用了 long double 也无法达到精度。因此可以考虑使用消除了最大公因数的斜率 (y, x) 对来当作哈希表的键值，键值对应的 value 为这个斜率的直线上包含的所有元素。

同时，为了解决可能出现的重合点的问题，需要在内层循环就进行一个判断，如果是重合的点就 ++samepoints， 如果不是的话通过：

用当前斜率下的直线上的点的个数值来更新 curmax 的值，再在外层循环内用 curmax + samepoints 来更新最大的结果。这里要琢磨一下，其实 samepoints 这里面的个数是与外层循环的每一个 (p1_x, p1_y) 相等的点的个数，必然可以加到从 (p1_x, p1_y) 出发的直线上。

146. LRU Cache/LRU缓存机制

题目：

运用你所掌握的数据结构，设计和实现一个 LRU (最近最少使用) 缓存机制。它应该支持以下操作： 获取数据 get 和 写入数据 put 。

获取数据 get(key) - 如果密钥 (key) 存在于缓存中，则获取密钥的值（总是正数），否则返回 -1。
写入数据 put(key, value) - 如果密钥已经存在，则变更其数据值；如果密钥不存在，则插入该组「密钥/数据值」。当缓存容量达到上限时，它应该在写入新数据之前删除最久未使用的数据值，从而为新的数据值留出空间。

进阶:

你是否可以在 O(1) 时间复杂度内完成这两种操作？

示例:

解法：⭐⭐⭐⭐⭐哈希表+双向链表=双向哈希表

这里面使用了双向链表 list 数据结构的 iterator 迭代器。

list 是一个线性双向链表结构，它的数据由若干个节点构成，每一个节点都包括一个信息块（即实际存储的数据）、一个前驱指针和一个后驱指针。它无需分配指定的内存大小且可以任意伸缩，这是因为它存储在非连续的内存空间中，并且由指针将有序的元素链接起来。

list 可以使用 list<T> :: iterator testiterator; 的形式来构造一个迭代器指针。对于迭代器指针指向的数值可以用 *testiterator 来求得。

整体思路

分析上面的操作过程，要让 put 和 get 方法的时间复杂度为 O(1) ，我们可以总结出 cache 这个数据结构必要的条件：查找快，插入快，删除快，有顺序之分。

显然 cache 必须有顺序之分，以区分最近使用的和久未使用的数据；而且我们要在 cache 中查找键是否已存在；如果容量满了要删除最后一个数据；每次访问还要把数据插入到队头。

哈希表的特点是查找快，但是数据无固定顺序；链表有顺序之分，插入删除快，但是查找慢。所以结合一下，形成一种新的数据结构：哈希链表。

为什么要双向链表呢？

因为我们需要删除操作。删除一个节点不光要得到该节点本身的指针，也需要操作其前驱节点的指针，而双向链表才能支持直接查找前驱，保证操作的时间复杂度 O(1) 。

为什么要在链表中同时存储 key 和 val，而不是只存储 val？

因为当缓存容量已满，我们不仅仅要删除最后一个 Node 节点，还要把 map 中映射到该节点的 key 同时删除，而这个 key 只能由 Node 得到。如果 Node 结构中只存储 val ，那么我们就无法得知 key 是什么，就无法删除 map 中的键，造成错误。

具体实现的过程中:

对于每次 get 操作，很简单，只需要判断一下 get 的这个 key 是不是存在在哈希链表中：如果不存在直接返回 -1 ；如果存在的话，map[key] 是一个 list 迭代器指针的数据，可以通过 *map[key] 来取到对应的 (key, value) 键值对，然后用 cache.erase() 清除掉当前的迭代器指针，再把这个 (key, value) 键值对 push_front() 放到双向哈希链表的前端。记得放了后要把 map[key] 也相应更新，最后返回 value 。

对于每次 put 操作，大体上分两种：

  1. put 对应的 key 已经存在，需要完成变更 value & 添加到头的操作；
  2. put 对应的 key 不存在，需要完成加入 (key, value) 键值对的操作。同时需要以下讨论：
    1. cache 满了：需要完成清理最近最少使用的 cache 同时删除 map 的操作。
    2. cache 未满：直接添加到头。

给定一个包含 n + 1 个整数的数组 nums，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

示例 2:

说明：

  1. 不能更改原数组（假设数组是只读的）。
  2. 只能使用额外的 O(1) 的空间。
  3. 时间复杂度小于 O(n2) 。
  4. 数组中只有一个重复的数字，但它可能不止重复出现一次。

解法一：⭐⭐⭐双指针

跟 Leetcode 141 142 相似，这道题居然也可以用双指针来做我是没想到的。

在这道题中，我们可以把数组看成是一个链表，其中数组的下标就是指向元素的指针，把数组的元素也看作指针。如0是指针，指向nums[0]，而nums[0]也是指针，指向nums[nums[0]]。

这样我们就可以用到之前 Leetcode 142 中判断链表成环的位置的方法：

假设有这样一个样例：[1,2,3,4,5,6,7,8,9,5]。如果我们按照上面的循环下去就会得到这样一个路径: 1 2 3 4 5 [6 7 8 9] [6 7 8 9] [6 7 8 9] . . .这样就有了一个环，也就是6 7 8 9。point会一直在环中循环的前进。
这时我们设置两个一快(fast)一慢(slow)两个指针，一个每次走两步，一个每次走一步，这样让他们一直走下去，直到他们在重复的序列中相遇。

解法二：⭐⭐⭐⭐⭐二分法+抽屉原理

以 [1, 2, 2, 3, 4, 5, 6, 7] 为例，一共 8 个数，n + 1 = 8 ，n = 7，根据题目意思，每个数都在 1 和 7 之间。

例如：区间 [1, 7] 的中位数是 4，遍历整个数组，统计小于等于 4 的整数的个数，至多应该为 4 个。换句话说，整个数组里小于等于 4 的整数的个数如果严格大于 4 个，就说明重复的数存在于区间 [1, 4]，它的反面是：重复的数存在于区间 [5, 7] 。

于是，二分法的思路是先二分地猜一个数（有效范围 [left, right] 里的中间数 mid），然后统计原始数组中小于等于这个中间数的元素的个数 cnt，如果 cnt 严格大于 mid，依然根据抽屉原理，重复元素就应该在区间 [left, mid] 里。

进阶：
你是否可以不用额外空间解决此题？

解法：⭐⭐⭐双指针

这道题跟上一道题一样是很经典的快慢指针问题，不过这道题需要对如何判断环的入口进行一个计算和判断：

  1. 双指针第一次相遇： 设两指针 fast，slow 指向链表头部 head，fast 每轮走 2 步，slow 每轮走 1 步；

    1. 第一种结果： fast 指针走过链表末端，说明链表无环，直接返回 null；
    2. 若有环，两指针一定会相遇。因为每走 1 轮，fast 与 slow 的间距 +1，fast 终会追上 slow；
      第二种结果： 当 fast == slow 时， 两指针在环中第一次相遇 。下面分析此时 fast 与 slow 走过的 步数关系 ：

  2. 设链表共有 a+b 个节点，其中链表头部到链表入口有 a 个节点（不计链表入口节点，链表环有 b 个节点；设快慢指针分别走了 f，s 步，则有：
    fast 走的步数是 slow 步数的 2 倍，即 f = 2s；
    fast 比 slow 多走了 n 个环的长度，即 f = s + nb；
    以上两式相减得：f = 2nb，s = nb，即 fast 和 slow 指针分别走了 2n，n 个 环的周长 （注意： n 是未知数，不同链表的情况不同）。

  3. 如果让指针从链表头部一直向前走并统计步数 k， 那么所有 走到链表入口节点时的步数 是：k=a+nb。
    而目前，slow 指针走过的步数为 nb 步。因此，我们只要想办法让 slow 再走 a 步停下来，就可以到环的入口。依然是使用双指针法。我们构建一个指针，此指针需要有以下性质：此指针和 slow 一起向前走 a 步后，两者在入口节点重合。那么从哪里走到入口节点需要 a 步？答案是链表头部 head 。

  4. slow 指针 位置不变 ，将 fast 指针重新指向链表头部节点；slow 和 fast 同时每轮向前走 1 步；此时 f = 0，s = nb ；当 fast 指针走到 f = a 步时，slow 指针走到步 s = a+nb ，此时 两指针重合，并同时指向链表环入口 。返回 slow 指针指向的节点。

❗ 注意，这里面的 fast 比 slow 多走了 n 个环的长度并不是说在环里的长度，而是两者走过的总长度相差了 n 个环的长度，想象一下环前面的直路很长而环很短的情况。

解法二：⭐⭐⭐记忆化回溯

拿 [aaaab] 来举例，它可分为 ["a","a","aab"] , ["aa","aab"] , 两个都包含了 "aab" ,所以就产生了重复的递归。这时，就可以用一个哈希表记录 "aab" 这种情况。当再次递归时，先判断一下，如果是 "aab" ,就直接返回。

解法三：⭐⭐⭐⭐动态规划+贪心思想

  1. 定义 vector<bool> dp(s.size() + 1, false)
  2. dp[i] 表示第 i 个字符前，所有的字符是否可以按字典拆分成单词；其中第 i - 1 个字符是上一个词的结尾字母，第 i 个字符是一个新词的开头字母。
  3. 初始值 dp[0] = true ，表示第 0 个字符是一个新词的开头字母；所求的答案为 dp[s.size()] ，表示这个字符串是否可以拆分成字典中的单词。
  4. 从前往后遍历字符串，我们只对新词的开头字母进行字典匹配，将每一个匹配成功的词后面的字母更新为新词开头字母。
  5. 设置一个变量记录所有最大的新词开头字母，如果当前字符已经超过了所有新词开头字母，可以剪枝返回 false 了。

这个解法里面用到了 C++ 的 memcmp 库函数：

库函数 int memcmp(const void *str1, const void *str2, size_t n)) 把存储区 str1 和存储区 str2 的前 n 个字节进行比较。

  • str1 – 指向内存块的指针。
  • str2 – 指向内存块的指针。
  • n – 要被比较的字节数。
  • 如果返回值 < 0，则表示 str1 小于 str2。
  • 如果返回值 > 0，则表示 str2 小于 str1。
  • 如果返回值 = 0，则表示 str1 等于 str2。
