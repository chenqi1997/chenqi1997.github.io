题目：

给定一个整数 n，生成所有由 1 … n 为节点所组成的二叉搜索树。

示例:

解法一：递归

构造一个 dfs 函数，在每一次调用的时候遍历取区间内的值作为中间节点，然后依次对于左半边和右半边的序列调用 dfs 函数，从而使得对于每一个中间的值 i ，都能够遍历到它左右子树的所有取值方式。同时这样的递归取法也可以保证构造的是二叉搜索树，严格遵循了比根节点小的在左边，大的在右边。

所以如果求 1…n 的所有可能。

我们只需要把 1 作为根节点，[ ] 空作为左子树，[ 2 … n ] 的所有可能作为右子树。

2 作为根节点，[ 1 ] 作为左子树，[ 3…n ] 的所有可能作为右子树。

3 作为根节点，[ 1 2 ] 的所有可能作为左子树，[ 4 … n ] 的所有可能作为右子树，然后左子树和右子树两两组合。

4 作为根节点，[ 1 2 3 ] 的所有可能作为左子树，[ 5 … n ] 的所有可能作为右子树，然后左子树和右子树两两组合。

…

n 作为根节点，[ 1… n ] 的所有可能作为左子树，[ ] 作为右子树。

94. Binary Tree Traversal/二叉树的遍历

题目：

Leetcode 94/144/145

给定一个二叉树，返回它的 前序、中序、后序 遍历。

示例:

进阶: 递归算法很简单，你可以通过迭代算法完成吗？

前序遍历：根->左->右

递归做法：

递归的思想很简单，就是先访问根节点，然后依次访问左子树和右子树。

⭐⭐⭐迭代做法：

迭代的主要做法是通过入栈来实现节点的遍历，每到一个节点 A，就应该立即访问它。因为，每棵子树都先访问其根节点。对节点的左右子树来说，也一定是先访问根。在 A 的两棵子树中，遍历完左子树后，再遍历右子树。因此，在访问完根节点后，遍历左子树前，要将右子树压入栈。

中序遍历：左->根->右

递归解法：

⭐⭐⭐迭代解法：

思路：每到一个节点 A，因为根的访问在中间，将 A 入栈。然后遍历左子树，接着访问 A，最后遍历右子树。

在访问完 A 后，A 就可以出栈了。因为 A 和其左子树都已经访问完成。

后序遍历：左->右->根

递归解法：

⭐⭐⭐迭代解法一

可以把后序遍历的 左->右->根 在每一次迭代的过程中取逆为 根->右->左 ，其实就跟前序遍历的迭代方式差不多了，只不过最后遍历结束需要加一个 reverse 。

⭐⭐⭐迭代解法二

还是正常按照左->右->根 的方式来进行遍历，当左子树和右子树都访问过后返回到该节点 A 即可访问。但是有一个问题是在迭代的过程中是无法判断出来迭代回来的是从左子树还是右子树返回的，因此需要对每一个节点 A 加一个哈希表来判断是否是从左子树返回的，当访问到 A 节点的右子树时 flag 赋值为 1。当哈希表中的 flag 为 0 时说明是从左子树返回的，为 1 时说明是从右子树返回的。

90. Subsets II/子集 II

题目：

给定一个可能包含重复元素的整数数组 nums，返回该数组所有可能的子集（幂集）。

说明：解集不能包含重复的子集。

示例:

解法一：DFS+剪枝回溯

这个题跟前面的 Leetcode 78 子集 总体思路是一样的，只不过加上了可能重复的元素，面对这样的条件也是很套路化了，先 sort 一遍，然后按照正常的回溯法来依次加到 tmp 里，再把 tmp 依次加到 res 里。

⭐⭐⭐整个代码最关键的一行在于 if(i > num && nums[i] == nums[i - 1]) continue;

这一行里面重点在于 i > num 这一个剪枝判断方法，这一个的目的是可以用 num 作为区分重复元素应不应该放进 tmp 的标准。i > num 就 continue 可以保证第一个 i = num 的时候可以取到重复元素的第一个元素并正常push 作为头节点。也就是说剪枝是从每一个递归层的第二层开始的，所以加上 i > num 。

  2. 给定矩阵中至少有一个元素是 0。
  3. 矩阵中的元素只在四个方向上相邻: 上、下、左、右。

解法一：动态规划

最开始想到的方法是动态规划，不过这里面题目中每一个点的值其实是取值与上下左右四个点的，而一般的动态规划只能够从左上到右下进行一次遍历，这样得到的结果是缺少了右侧和下方的数值的，因此可以想到用两次动态规划遍历同一个 dp 数组，第二次从右下向左上遍历。

解法二：BFS

相当于把所有的 0 看成是一个整体，存在一个 超级零 与所有的0相连，相当于最短路算法中的 超级远点 ，连向所有的源点，用 超级源点 到终点的最短路等价多个源点到终点的终点。在起始的时候把所有的位置为 0 的点都放入队列中，然后不断广播更新 result 二维矩阵中的值：如果当前广播到的 result[x][y] 的值大于当前队首坐标的值 + 1 ，那么就可以用这个队首坐标的值 + 1来代替当前的 result。同时将这个目前广播到的(x, y) 入队，进行更深层次的广播。
