题目：

给定一个二维网格和一个单词，找出该单词是否存在于网格中。

单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母不允许被重复使用。

示例:

提示：

  • board 和 word 中只包含大写和小写英文字母。
  • 1 <= board.length <= 200
  • 1 <= board[i].length <= 200
  • 1 <= word.length <= 10^3

解法：DFS+回溯

这道题跟以往的回溯法的题目其实差不多，构造的 backTrack 函数结构也是类似的，不过这里面可以构造一个 bool 类型的 backTrack 函数，因为需要在每一步的计算过程中都判断当前深度的四个方向上是否是可以继续走下去的，如果四个方向都是 false ，那么就需要进行回溯，把 board[x][y] 恢复到上一个状态。这里面可以通过将 board[x][y]置零的方法来避免走到重复的格子，也可以省去了哈希表的空间。

解法：⭐⭐⭐⭐⭐动态规划+二分法

这道题太难了，第一遍想的时候其实想到了二分法，也考虑可能会用动态规划的思想，不过状态转移方程没想出来-。-

这里面的状态即为 (K, N) ，代表着当前有 K 个鸡蛋， N个楼层数。dp(K, N) 为在 (K, N) 状态下所需要的最少步数，这里面可以对 dp(K, N) 的状态转移方程进行一个讨论：

  1. 这个状态鸡蛋没碎：那么状态变成 (K,N−X)，即我们鸡蛋的数目不变，但答案只可能在上方的 N−X层楼了。也就是说，我们把原问题缩小成了一个规模为(K,N−X) 的子问题；
  2. 这个状态鸡蛋碎了：那么状态变成 (K−1,X−1)，即我们少了一个鸡蛋，但我们知道答案只可能在第 X 楼下方的 X−1 层楼中了。也就是说，我们把原问题缩小成了一个规模为 (K−1,X−1) 的子问题。

如果按照常规思路来遍历计算每一个 dp(K, N) 的值再去得到最小的值，时间复杂度为 O(K * N^2) ，会超时，那么我们可以进一步考虑，由于 dp(K, N) 是一个关于 N 的单调递增函数，即对于一个固定的 K 个鸡蛋的情况，N 值越大，最终需要的最少步数一定也越大。因此我们可以通过二分法来优化 O(logN) 的时间复杂度。而在 dp(K, N) 的定义式里面，T1(X) = dp(K-1, X-1) 是关于 X 单调增的，T2(X) = dp(K, N-X) 是关于 x 单调递减的。

可以通过二分+判断的方式使得我们二分法的 lo 最后到达 x0 的位置， hi 到达 x1 的位置。这里面由于 T1(X)和 T2(X) 是离散函数，不一定能取到相交点的值，我们只能通过不断逼近的方式来计算。最终我们再去比较这两个值里面的最小值，即为我们最终的答案。

我们在所有满足条件的 X 上进行二分查找。对于状态 (K, N) 而言，X即为 [1, N] 中的任一整数；

在二分查找的过程中，假设当前这一步我们查找到了 Xmid，如果 T1(Xmid) > T2(Xmid)，那么真正的 X_0 一定在 Xmid 的左侧，否则真正的 X_0 在 Xmid 的右侧。

76. Minimum Window Substring/最小覆盖子串

题目：

给你一个字符串 S、一个字符串 T，请在字符串 S 里面找出：包含 T 所有字母的最小子串。

示例：

说明：

  • 如果 S 中不存这样的子串，则返回空字符串 ""。
  • 如果 S 中存在这样的子串，我们保证它是唯一的答案。

解法：⭐⭐⭐⭐双指针+滑窗

滑动窗口算法的思路是这样：

  1. 我们在字符串 S 中使用双指针中的左右指针技巧，初始化 left = right = 0，把索引闭区间 [left, right] 称为一个「窗口」。
  2. 我们先不断地增加 right 指针扩大窗口 [left, right]，直到窗口中的字符串符合要求（包含了 T 中的所有字符）。
  3. 此时，我们停止增加 right，转而不断增加 left 指针缩小窗口 [left, right]，直到窗口中的字符串不再符合要求（不包含 T 中的所有字符了）。同时，每次增加 left，我们都要更新一轮结果。
  4. 重复第 2 和第 3 步，直到 right 到达字符串 S 的尽头。

这个思路其实也不难，第 2 步相当于在寻找一个「可行解」，然后第 3 步在优化这个「可行解」，最终找到最优解。左右指针轮流前进，窗口大小增增减减，窗口不断向右滑动。

75. Sort Colors/颜色分类

题目：

给定一个包含红色、白色和蓝色，一共 n 个元素的数组，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

此题中，我们使用整数 0、 1 和 2 分别表示红色、白色和蓝色。

注意:
不能使用代码库中的排序函数来解决这道题。

示例:

进阶：

  • 一个直观的解决方案是使用计数排序的两趟扫描算法。
    首先，迭代计算出0、1 和 2 元素的个数，然后按照0、1、2的排序，重写当前数组。
  • 你能想出一个仅使用常数空间的一趟扫描算法吗？

解法：双指针

这里面的策略是用两个指针 left 和 right 来维护当前已经扫描过的数组的部分中 0 的右侧边界的下一个元素和 2 的左侧边界的上一个元素。在遍历 cur 的过程中，通过判断 cur 的值来进行交换。

❗ 注意，这里面 if(nums[cur] == 0) swap(nums[cur++], nums[left++]); cur 是已经遍历过 left 的位置的，就不需要交换之后再判断一次 cur 的值了，因此可以 nums[cur++] 。而与后面的交换中，不能 cur++ ，因为从后面换过来的元素是未遍历过的，不知道是 0 还是 1 还是 2，需要把 cur 保存在原地再遍历一下。
