J04.二维数组中的查找

题目：

在一个 n * m 的二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

示例:

现有矩阵 matrix 如下：

[
[1, 4, 7, 11, 15],
[2, 5, 8, 12, 19],
[3, 6, 9, 16, 22],
[10, 13, 14, 17, 24],
[18, 21, 23, 26, 30]
]
给定 target = 5，返回 true。

给定 target = 20，返回 false。

限制：

0 <= n <= 1000

0 <= m <= 1000

解法

这道题既然都已经排了序，就不要考虑暴力解法O(mn)遍历数组中每一个元素了，如果从左上角开始比较的话，对于该向右走还是向下走根本不能判断。观察矩阵的特点可以发现，对于左下角和右上角的元素特殊性比较大，从左下角开始，往上走是比它小的，往右走是比它大的，因此可以很容易用 while(true) 循环遍历完直到找到对应的值。注意，如果找不到对应的值的条件应该是在多次循环之后”破了壁”，即走出了矩阵之外，否则都有可能会找到这个值。

543. Diameter of Binary Tree/二叉树的直径

题目

给定一棵二叉树，你需要计算它的直径长度。一棵二叉树的直径长度是任意两个结点路径长度中的最大值。这条路径可能穿过根结点。

示例 :
给定二叉树

返回 3, 它的长度是路径 [4,2,1,3] 或者 [5,2,1,3]。

注意：两结点之间的路径长度是以它们之间边的数目表示。

解法

分析一下，题目中写的是要求一棵二叉树的直径长度，乍一看是求根节点左右两个子树的深度之和。但是题目中说到了，二叉树的直径不一定过根节点。

二叉树的解题技巧是，首先判断问题能否划分为子问题、应当划分为什么样的子问题。二叉树直径实际上就是二叉树中的最长路径，我们是可以划分出子问题的：

二叉树的最长路径=max{左子树的最长路径,右子树的最长路径,经过根结点的最长路径}

其中左子树的最长路径和右子树的最长路径是两个可以递归求解的子问题，那么经过根结点的最长路径如何计算呢？是左子树的深度加上右子树的深度。代入上面的式子得到：
二叉树的最长路径=max{左子树的最长路径,右子树的最长路径,左子树的深度+右子树的深度}

因此这道题的算法思路便是通过构造递归函数，从根节点对二叉树进行遍历。对于每一次递归，我们先计算左子树的maxDepth()，在计算右子树的maxDepth，那么对于这个节点来说，只需要求出这个节点左右子树深度之和，作为一个候选值来和根节点的maxDepth()值进行比较，通过左右子树向下的不断遍历更新这个全局变量res，直到获得最好的结果res。

考虑到遍历过程中可能对同一个节点重复计算多次深度，为了减少计算量，定义一个哈希表unordered_map<TreeNode*, int> m;来保存每个节点和深度的映射，这样如果在计算的过程中遇到了相同的节点，可以直接return 这个节点的深度。

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。
注意空字符串可被认为是有效字符串。

示例 1:

输入: “()”
输出: true

示例 2:

输入: “()[]{}”
输出: true

示例 3:

输入: “(]”
输出: false

示例 4:

输入: “([)]”
输出: false

示例 5:

输入: “{[]}”
输出: true

解法

这个是之前做过的一个很经典的用到栈的题，主要逻辑是利用栈顶的push() top()与 pop()函数来对 s[i] 元素进行操作，如果是左括号( [ { 则依次 push() 到栈里面，如果是右括号就 push栈顶的元素看是不是匹配，如果匹配就用 pop() 函数来删除栈顶的左括号，如果不匹配直接return false。

⭐复习有关stack STL的函数如push / top / pop / empty / size 等等。

16. Closest Three Sum/最接近的三数之和

题目：

给定一个包括 n 个整数的数组 nums 和 一个目标值 target。找出 nums 中的三个整数，使得它们的和与 target 最接近。返回这三个数的和。假定每组输入只存在唯一答案。

例如，给定数组 nums = [-1，2，1，-4], 和 target = 1.与 target 最接近的三个数的和为 2. (-1 + 2 + 1 = 2).

解法：

这道题跟前面的 15.Three Sum 的思想很像，还是用了双指针的思想，先对数组进行sort排序，然后遍历一个 i 来从左至右进行扫描，在遍历 i 的过程中，j 和 k 两个指针仍然是从左右各自向中间逼近。与 Three Sum不同的是这次没有一个明确的目标值 0，而是要找与目标值target差最小的值。这里我们在循环中维护一个 sum = nums[i] + nums[j] + nums[k]，最开始定义一个很大的变量 val用来存储 sum 和 target在当前的最小误差，如果当前的最小误差比循环中得到的 tmp 误差还要大，那么就把新得到的 tmp 存入val中，同时把当前的 sum 存入 closest当中。最后可以加一个很松的判断中止的条件，如果nums[i] 已经大于等于 target / 3，那么就没必要再循环了，可以直接返回closest和nums[i]后面连续三个数相加的和中最小的：if (nums[i] * 3 > target) return min(closest, nums[i] + nums[i + 1] + nums[i + 2]);

发表于 2020-03-08 | 分类于 LeetCode ， Medium

15. Three Sum/三数之和

题目：

给你一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？请你找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例：

给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
[-1, 0, 1],
[-1, -1, 2]
]

解法

为了防止答案中包含重复的三元组，在固定一个数来求two sum问题的时候，一定要保证遍历的顺序性。这里面先用 sort() 函数来对整个数组进行排序，得到一个递增的有序数列；然后再通过每次都固定遍历最左边的元素作为 0 - nums[k] 中的 nums[k]， 然后在 k + 1到 nums.size() - 1的区间内利用双指针进行 two sum问题求解，注意这里面i + 1到 nums.size() - 1的区间内是有序的数组，因此只需要判断双指针的数值和与 0 - nums[k] 的大小关系来移动指针就可以；同时为了防止出现重复，在遍历 k 的时候，要判断一下是否有重复的数，如果重复直接continue，如果 nums[k]已经大于零了就可以直接 break，三者的和必大于零；

⭐同时在求two sum问题的时候也要注意，这个区间内的有序数组也可能是有重复的元素，需要在双指针移动之后进行一个判断，如果指针和下一个移动点的值是重复的，就要做一次移动来避免向res存入重复的结果。

发表于 2020-03-08 | 分类于 LeetCode ， Easy

14. Longest Common Prefix/最长公共前缀

题目

编写一个函数来查找字符串数组中的最长公共前缀。如果不存在公共前缀，返回空字符串 ""。

示例 1:

输入: [“flower”,”flow”,”flight”]
输出: “fl”

示例 2:

输入: [“dog”,”racecar”,”car”]
输出: “”
解释: 输入不存在公共前缀。

解法一

思路比较简单，为了减少运算量，先找出字符串数组中最小的字符串的长度，然后以这个最小长度为循环次数，依次从左到右遍历字符串数组中每一个字符串对应位置的字符来进行比较，如果有不相等的情况直接 return 当前的 res ，如果遍历一遍没有不同的情况就 res += s，再继续遍历。

解法二

巧妙解法：这种方法给输入字符串数组排了个序，想想这样做有什么好处？

既然是按字母顺序排序的话，那么有共同字母多的两个字符串会被排到一起，而跟大家相同的字母越少的字符串会被挤到首尾两端，那么如果有共同前缀的话，一定会出现在首尾两端的字符串中，所以只需要找首尾字母串的共同前缀即可。

比如例子1排序后为 [“flight”, “flow”, “flower”]，例子2排序后为 [“cat”, “dog”, “racecar”]，虽然例子2没有共同前缀，但也可以认为共同前缀是空串，且出现在首尾两端的字符串中。由于是按字母顺序排的，而不是按长度，所以首尾字母的长度关系不知道，为了防止溢出错误，只遍历而这种较短的那个的长度，找出共同前缀返回即可，在每次比较的时候也不用依次比较了。
