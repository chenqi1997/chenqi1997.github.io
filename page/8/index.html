J28. 对称的二叉树 / Leetcode 101

题目：

请实现一个函数，用来判断一棵二叉树是不是对称的。如果一棵二叉树和它的镜像一样，那么它是对称的。

例如，二叉树 [1,2,2,3,4,4,3] 是对称的。

    1
   / \
  2   2
 / \ / \
3  4 4  3

但是下面这个 [1,2,2,null,3,null,3] 则不是镜像对称的:

  1
 / \
2   2
 \   \
 3    3

示例 1：

输入：root = [1,2,2,3,4,4,3]
输出：true

示例 2：

输入：root = [1,2,2,null,3,null,3]
输出：false

限制：

0 <= 节点个数 <= 1000

解法一：利用J27里面的 mirrorTree（太笨了！！）

很烦，刚做完27题所以想着这道题不如就先写一个 mirrorTree得到对称之后的树，再写一个 equalTree 来判断一下是否相等就行，但是明明感觉逻辑没什么问题但是代码就是会错，排查了半天也没排查明白，先占个坑TAT

解法二：递归思想

如果递归来看，对于目前的手上的两个节点 p 和 q，对称即为 p 的左子节点的值和 q 的右子节点的值相等，且 p 的右子节点的值与 q 的左子节点的值相等。这里面为了方便递归，我们先给出 root 的左右节点，将它们分别作为新定义的重载函数 isSymmetric 的输入。

解法三：迭代思想/BFS

利用队列来进行迭代，每次提取两个节点并且比较它们的值，然后同时将两个节点的左右子节点按相反的顺序插入到队列中，直到队列中不存在元素或者树不对称时结束。

J27. 二叉树的镜像 / Leetcode 226

题目：

请完成一个函数，输入一个二叉树，该函数输出它的镜像。

例如输入：

     4
   /   \
  2     7
 / \   / \
1   3 6   9

镜像输出：

     4
   /   \
  7     2
 / \   / \
9   6 3   1

示例 1：

输入：root = [4,2,7,1,3,6,9]
输出：[4,7,2,9,6,3,1]

限制：

0 <= 节点个数 <= 1000

解法一：递归法

遇到树最先想到的想法是做递归，通过不断进行递归向下完成树的从上到下的旋转。在每一层的旋转过程中都借助一个 tmp 指针来交换传递左右指针，最后返回根节点。

⭐解法二：非递归法：队列实现

递归的实现方法可以通过队列的先入先出性质来实现，在遍历节点的过程中，先放入根节点 root，再取出头节点并交换左右节点，如果左右节点存在，再存入队列中并且依次取出，从而不断向下遍历。

注意 queue 队列类没有 top 函数，有的是 front , back , pop , empty , size 。不要跟栈弄混了。

J26. 树的子结构

题目：

输入两棵二叉树A和B，判断B是不是A的子结构。(约定空树不是任意一个树的子结构)

B是A的子结构， 即 A中有出现和B相同的结构和节点值。

例如:
给定的树 A:

    3
   / \
  4   5
 / \
1   2

给定的树 B：

返回 true，因为 B 与 A 的一个子树拥有相同的结构和节点值。

示例 1：

输入：A = [1,2,3], B = [3,1]
输出：false

示例 2：

输入：A = [3,4,5,1,2], B = [4,1]
输出：true

限制：

0 <= 节点个数 <= 10000

解法：

如果树 B 是树 A 的子结构，那么树 B 根节点一定是树 A 的某一个子节点，这里面对于二叉树的遍历分为两步：

  1. 先序遍历树 A 中的每个节点，递归调用 isSubStructure 函数，来判断当前 A 的节点是否为 B 的根节点。
  2. 如果已经是 B 的根节点了，递归调用 recur 函数来进行 DFS 搜索，在搜索过程中判断 B 是否与找到的 A 的子树有相同的结构和节点值，这里面搜索的过程判断的应该是 当前节点值相等，再调用左右子树做 DFS 搜索。

  1. 如果 B 为空，则说明 B 已经全部匹配完成，应该返回 true 。
  2. 再判断 A 是否为空，如果 B 非空而 A 为空，则说明 A 的这部分子树匹配不上 B，应该返回 false。
  3. 再判断 A 和 B 均非空时，对应的节点值是否相同，如果相同才可以继续递归向下。

J21. 调整数组顺序使奇数位于偶数前面

题目：

输入一个整数数组，实现一个函数来调整该数组中数字的顺序，使得所有奇数位于数组的前半部分，所有偶数位于数组的后半部分。

示例：

提示：

  1. 1 <= nums.length <= 50000
  2. 1 <= nums[i] <= 10000

解法一：用栈

这道题看起来比较简单，但是麻烦的是取出来偶数之后，数组中剩下的奇数元素还需要通过遍历才能放到前半段，想用类似 冒泡排序 的做法进行奇数冒泡但是考虑到O(n^2)的复杂度放弃了。因此为了降低时间复杂度，想到利用栈的结构来进行奇数偶数的分别存储，再放回原数组，这样时间空间复杂度均为O(n)。写出来代码之后万万没想到这种解法居然内存消耗击败100%，时间26.95%。人算不如天算啊-。-

解法二：⭐⭐⭐双指针法

双指针法的好处是可以直接对于奇数和偶数的 index 进行记录，方便对于奇数和偶数的交换。这样也就不用再申请额外的空间了，得到的时间复杂度为O(n)，空间复杂度为O(1)。

双指针法在这里还可以分为两种，思想上大同小异：

快慢指针法

j保存当前待交换的位置， i向后滑动寻找奇数，找到后交换 nums[i] 和 nums[j] , 每次交换后j向后滑动1，[j,i)之间绝对不会出现奇数。当 i 遍历到 n - 1时结束。

剑指offer_20 表示数值的字符串 / Leetcode 65

J20. 表示数值的字符串 / Leetcode 65

题目：

请实现一个函数用来判断字符串是否表示数值（包括整数和小数）。例如，字符串”+100”、”5e2”、”-123”、”3.1416”、”0123”及”-1E-16”都表示数值，但”12e”、”1a3.14”、”1.2.3”、”+-5”及”12e+5.4”都不是。

解法：讨论

吸取了别人blog里面比较优雅的讨论方法，这道题可以把所有的字符可以分为六大类：

空格，符号，数字，小数点，自然底数和其他字符。

我们需要五个标志变量，num, dot, exp, sign 分别表示数字，小数点，自然底数和符号是否出现，numAfterE表示自然底数后面是否有数字，那么我们分别来看各种情况：

- 空格： 我们需要排除的情况是，当前位置是空格而后面一位不为空格，但是之前有数字，小数点，自然底数或者符号出现时返回false。

- 符号：符号前面如果有字符的话必须是空格或者是自然底数，标记sign为true。

- 数字：标记 num和numAfterE为true。

- 小数点：如果之前出现过小数点或者自然底数，返回false，否则标记dot为true。

- 自然底数：如果之前出现过自然底数或者之前从未出现过数字，返回false，否则标记exp为true，numAfterE为false。

- 其他字符：返回false。

最后返回num && numAfterE即可。
