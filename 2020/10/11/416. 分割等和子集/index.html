416. 分割等和子集

题目：

给定一个只包含正整数的非空数组。是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

注意:

  1. 每个数组中的元素不会超过 100
  2. 数组的大小不会超过 200

解法一：二维DP

这道题刚开始完全没思路，后来看了官方题解发现是NP完全问题，根本没办法多项式范围内求解。

正因如此，我们不应期望该问题有多项式时间复杂度的解法。我们能想到，例如基于贪心算法的「将数组降序排序后，依次将每个元素添加至当前元素和较小的子集中之类的方法都是错误的，可以轻松地举出反例。因此，我们必须尝试非多项式时间复杂度的算法，例如时间复杂度与元素大小相关的动态规划。

这道题可以换一种表述：给定一个只包含正整数的非空数组nums[0]，判断是否可以从数组中选出一些数字，使得这些数字的和等于整个数组的元素和的一半，也就是下面的 target 。因此这个问题可以转换成0−1 背包问题。这道题与传统的0−1背包问题的区别在于，传统的0−1 背包问题要求选取的物品的重量之和不能超过背包的总容量，这道题则要求选取的数字的和恰好等于整个数组的元素和的一半。类似于传统的0−1 背包问题，可以使用动态规划求解。

dp定义

创建二维数组 dp ，包含 n 行 target + 1 列，其中 dp[i][j]表示从数组的 [0,i] 下标范围内选取若干个正整数（可以是 0 个），是否存在一种选取子集方案使得被选取的正整数的和等于 j。初始时，dp 中的全部元素都是 false 。

这样在后面的状态转移的过程中相当于是从小的 j 开始不断向更大的 j 来传递，最后题目要得到的结果是 dp[n - 1][target]，表示整个数组能否选取出一个子集使得和为 sum / 2 = target 。

状态转移方程：

由于 dp[i][j] 表示 [0, i] 能够有子集和为 j。那么在传递的过程中肯定要与 nums[i] 产生关系，我们可以进行一个讨论：

  1. 如果 j >= nums[i]， 说明目前的这个和可以从 nums[0, i - 2] 这个数组中的更低层次得来，也就是 dp[i - 1][j - nums[i]] 得来。这个代表的意思是我们可以取第 i - 1 个数字，也可以不取第 i - 1 个数字，因此状态转移方程为 ：dp[i][j] = dp[i - 1][j] || dp[i - 1][j - curNum]。
  2. 如果 j < nums[i]，说明无法取第 i - 1 个数字，因此状态转移方程为：dp[i][j] = dp[i - 1][j]。
