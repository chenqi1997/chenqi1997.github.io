220*. Contains Duplicate III/存在重复元素III

题目：

给定一个整数数组，判断数组中是否有两个不同的索引 i 和 j，使得 nums [i] 和 nums [j] 的差的绝对值最大为 t，并且 i 和 j 之间的差的绝对值最大为 ķ。

解法：⭐⭐⭐⭐⭐利用平衡二叉搜索树BST+Set实现

set 是 STL 中一种标准关联容器。它底层使用平衡的搜索树——红黑树实现，插入删除操作时仅仅需要指针操作节点即可完成，不涉及到内存移动和拷贝，所以效率比较高。set ，顾名思义是“集合”的意思，在 set 中元素都是唯一的，而且默认情况下会对元素自动进行升序排列，支持集合的交 (set_intersection) , 差 (set_difference) 、 并 (set_union) ，对称差 (set_symmetric_difference) 等一些集合上的操作，如果需要集合中的元素允许重复那么可以使用 multiset 。

总体思想：

  1. 初始化一颗空的二叉搜索树 set。
  2. 对于每个元素 x ，遍历整个数组。
  3. 在 set 上查找大于等于 x 的最小的数，如果s − x ≤ t 则返回 true 。
  4. 在 set 上查找小于等于 x 的最大的数，如果 x − g ≤ t 则返回 true 。
  5. 在 set 中插入x ，如果树的大小超过了 k , 则移除最早加入树的那个数。
  6. 返回 false

这里面 C++ 的 set::upper_bound和 Java 中的TreeSet::floor有点微妙的不同。为了遵循区间的左闭右开原则，set::upper_bound返回的是小于等于目标元素的最大元素的后一个元素（即大于目标元素的最小元素），所以要对返回的迭代器进行自减操作。

简单来说就是：

std::set::upper_bound 是 >
std::set::lower_bound 是 >=

解法二：⭐⭐⭐⭐⭐桶排序

我们换种思路。 由于本题对索引有要求，因此直接排序破坏了原来的索引的做法是不行的。 我们考虑使用桶排序。我们把窗口内的数字存在不同编号的桶中。每个桶内存的数字范围是 t + 1 个数，这样做的好处是，桶内任意两个数之间的差一定是小于等于 t 的。

  1. 我们将数据分到 M 个桶 中。
  2. 每个数字 nums[i] 都被我们分配到一个桶中。
  3. 分配的依据就是 nums[i] / (t + 1)。
  4. 这样相邻桶内的数字最多相差 2 * t + 1。
  5. 不相邻的桶一定不满足相差小于等于 t 。
  6. 同一个桶内共有 t + 1 个下标连续的数字，数字最多相差 t。
  7. 因此如果命中同一个桶内，那么直接返回 True 。
  8. 如果命中相邻桶，我们再判断一下是否满足 相差 <= t。
  9. 否则返回 False 。

需要注意的是，由于题目有索引相差 k 的要求，因此要维护一个大小为 k 的窗口，定期清除桶中过期的数字。在具体实现中使用哈希表来模拟桶， key 就是桶号，value 就是数字本身。时间复杂度为 O(N) 。
