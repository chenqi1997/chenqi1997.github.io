229*. Majority Elements II/求众数II

题目:

给定一个大小为 n 的数组，找出其中所有出现超过 ⌊ n/3 ⌋ 次的元素。

说明: 要求算法的时间复杂度为 O(n)，空间复杂度为 O(1)。

解法：⭐⭐⭐⭐⭐摩尔投票法

这道题跟 Leetcode 169 多数元素 差不多，只不过是169题里面只需要，找一个众数，那就只要一个候选，并且是保证一定有一个是众数的，直接投票就好；但是这个题没有保证有一个元素一定出现 n/3 以上，因此我们需要两次遍历。

摩尔投票法

摩尔投票法分为两个阶段：抵消阶段和计数阶段。

抵消阶段：两个不同投票进行对坑，并且同时抵消掉各一张票，如果两个投票相同，则累加可抵消的次数；

计数阶段：在抵消阶段最后得到的抵消计数只要不为 0 ，那这个候选人是有可能超过一半的票数的，为了验证，则需要遍历一次，统计票数，才可确定。

首先对于这样的问题，数学角度来看，个数超过n/k 的众数最多只有 k - 1 个。

因此我们可以在这道题里面先指定两个候选人 cand1 ，cand2 。然后不断遍历整个数组：

  1. 如果投 A （当前元素等于 A ），则 A 的票数++;
  2. 如果投 B （当前元素等于 B ）， 则 B 的票数++；
  3. 如果 A , B 都不投（即当前与 A ， B 都不相等）,那么检查此时 A 或 B 的票数是否减为 0 ，如果为 0 ，则当前元素成为新的候选人；如果 A , B 两个人的票数都不为 0 ，那么 A , B 两个候选人的票数均减一。此时说明是另外一个候选人，需要分别抵消掉 A 和 B 的一张选票。

这样抵消遍历投票一个循环下来，最后得到的两个候选人就是有可能投票超过 n / 3 的候选人，但是还需要一次计数遍历来判断是否投票真的超过了 n / 3 。
