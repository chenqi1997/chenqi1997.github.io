209. Minimum Size Subarray Sum/长度最小的子数组

题目:

给定一个含有 n 个正整数的数组和一个正整数 s ，找出该数组中满足其和 ≥ s 的长度最小的连续子数组，并返回其长度。如果不存在符合条件的连续子数组，返回 0。

示例:

进阶:

如果你已经完成了O(n) 时间复杂度的解法, 请尝试 O(n log n) 时间复杂度的解法。

解法一：⭐⭐⭐双指针

时间复杂度 O(N) ，因为每个元素最多被访问两次，right 一次，left 一次。

之前好像做过类似的题，但是想不起来是哪一道了😓首先想到的思路就是利用两个双指针 left 和 right 分别标记着符合条件的和 ≥ s 的连续子数组的左端点和右端点。总体的循环是在 while 中让 right 从 0 走到 nums.size() - 1 ，然后在循环中一旦满足了 cursum >= s ，就让 left 尽可能向右走，并且不断更新 res 的最小长度值。当 cursum 变得小于 s 了之后再继续让 right 向右走，以此往复直到 right 走到 nums.size() - 1 。

解法二：二分查找(迷惑)

  1. 创建大小为 n+1 的数组 sums ：sums[0] = 0, sums[i] = sums[i − 1] + nums[i − 1]

  2. 从 i=1 到 n 枚举：
    在 sums 中找到值 to_find ，满足从 i 开始到这个位置的和大于等于 s 且是最小子数组：
    to_find = s + sums[i − 1]

  3. 在 sums 中找到值满足大于等于 to_find 的下标，记作 bound

  4. 如果我们在 sums 中找到了值 to_find， 那么：

  • 当前子数组的大小为：bound − (sums.begin() + i − 1)
  • 将 ans 与当前数组的大小做比较，并把较小值保存到 ans 中。
