18. Four Sum/四数之和

题目

给定一个包含 n 个整数的数组 nums 和一个目标值 target，判断 nums 中是否存在四个元素 a，b，c 和 d ，使得 a + b + c + d 的值与 target 相等？找出所有满足条件且不重复的四元组。

注意：

答案中不可以包含重复的四元组。

示例：

给定数组 nums = [1, 0, -1, 0, -2, 2]，和 target = 0。

满足要求的四元组集合为：
[
[-1, 0, 0, 1],
[-2, -1, 1, 2],
[-2, 0, 0, 2]
]

解法

这道题的解法跟 15. Three Sum没有任何不同，只是加了一个循环。注意，这次的测试样例里面要考虑输入数组元素个数不足四个的情况。

❓奇怪的是，别人博客里面的解法是不用像我这样判断 nums.size() < 4的，就直接上来写一个 int n = nums.size()，在后面的循环里面也都相应的把nums.size()替换成 n，这样就能通过输入样例为 [] 0的测试；

别人家的代码如下：

如果把int n = nums.size()去掉，把循环里面对应的用nums.size()表示就会报以下错误：

执行出错信息：AddressSanitizer: SEGV on unknown address 0x000000000008 (pc 0x000000405d4f bp 0x7ffee5e58a30 sp 0x7ffee5e57e90 T0)

最后执行的输入：[] 0

不是很懂啊，如果按照 int n = nums.size()来看的话，输入[] 0，int n = nums.size() = 0，连第一个循环都进不去，直接返回没有返回值的res了。

解法二

解法二可以调用 STL 中的 TreeSet （详见C++基础分类 STL）来省去两个循环中判断是否有重复的过程，在执行用时上会有一点点降低。这里利用的是 TreeSet 的性质是不能有重复，如果新加入的数在 TreeSet 中原本就存在的话，插入操作就会失败，这样能很好的避免的重复项的存在。不过跟解法一都是O($n^3$)的时间复杂度。
