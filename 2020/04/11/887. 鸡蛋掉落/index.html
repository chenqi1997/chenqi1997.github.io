887*. Super Egg Drop/鸡蛋掉落

题目：

你将获得 K 个鸡蛋，并可以使用一栋从 1 到 N 共有 N 层楼的建筑。

每个蛋的功能都是一样的，如果一个蛋碎了，你就不能再把它掉下去。

你知道存在楼层 F ，满足 0 <= F <= N 任何从高于 F 的楼层落下的鸡蛋都会碎，从 F 楼层或比它低的楼层落下的鸡蛋都不会破。

每次移动，你可以取一个鸡蛋（如果你有完整的鸡蛋）并把它从任一楼层 X 扔下（满足 1 <= X <= N）。

你的目标是确切地知道 F 的值是多少。

无论 F 的初始值如何，你确定 F 的值的最小移动次数是多少？

解法：⭐⭐⭐⭐⭐动态规划+二分法

这道题太难了，第一遍想的时候其实想到了二分法，也考虑可能会用动态规划的思想，不过状态转移方程没想出来-。-

这里面的状态即为 (K, N) ，代表着当前有 K 个鸡蛋， N个楼层数。dp(K, N) 为在 (K, N) 状态下所需要的最少步数，这里面可以对 dp(K, N) 的状态转移方程进行一个讨论：

  1. 这个状态鸡蛋没碎：那么状态变成 (K,N−X)，即我们鸡蛋的数目不变，但答案只可能在上方的 N−X层楼了。也就是说，我们把原问题缩小成了一个规模为(K,N−X) 的子问题；
  2. 这个状态鸡蛋碎了：那么状态变成 (K−1,X−1)，即我们少了一个鸡蛋，但我们知道答案只可能在第 X 楼下方的 X−1 层楼中了。也就是说，我们把原问题缩小成了一个规模为 (K−1,X−1) 的子问题。

这里面需要理解的是这个，由于题目要求的是最坏情况下需要走的最少步数，因此需要对上面讨论得到的两个状态进行一个取 max 的操作，再去对这个关于 X 的函数进行优化取最小值。

如果按照常规思路来遍历计算每一个 dp(K, N) 的值再去得到最小的值，时间复杂度为 O(K * N^2) ，会超时，那么我们可以进一步考虑，由于 dp(K, N) 是一个关于 N 的单调递增函数，即对于一个固定的 K 个鸡蛋的情况，N 值越大，最终需要的最少步数一定也越大。因此我们可以通过二分法来优化 O(logN) 的时间复杂度。而在 dp(K, N) 的定义式里面，T1(X) = dp(K-1, X-1) 是关于 X 单调增的，T2(X) = dp(K, N-X) 是关于 x 单调递减的。

可以通过二分+判断的方式使得我们二分法的 lo 最后到达 x0 的位置， hi 到达 x1 的位置。这里面由于 T1(X)和 T2(X) 是离散函数，不一定能取到相交点的值，我们只能通过不断逼近的方式来计算。最终我们再去比较这两个值里面的最小值，即为我们最终的答案。

我们在所有满足条件的 X 上进行二分查找。对于状态 (K, N) 而言，X即为 [1, N] 中的任一整数；

在二分查找的过程中，假设当前这一步我们查找到了 Xmid，如果 T1(Xmid) > T2(Xmid)，那么真正的 X_0 一定在 Xmid 的左侧，否则真正的 X_0 在 Xmid 的右侧。
